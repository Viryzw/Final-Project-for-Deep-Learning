##### 我们要做什么？

在实际应用场景中，手写字迹往往会受到纸面污损、笔迹模糊、文字残缺等多种因素影响，导致传统识别算法的性能显著下降。然而，在公安刑侦、司法鉴定、档案数字化等关键领域，对手写字迹的稳定识别能力具有极高要求。因此，开发一种能够在复杂环境下依旧保持高鲁棒性的手写字迹识别模型具有重要的现实意义。

基于这一背景，我们的项目旨在构建一种具有特征选择能力与抗干扰能力的深度学习手写识别系统。具体来说，我们希望利用卷积神经网络（CNN）强大的图像表征能力提取笔画形态、结构布局等深层次视觉特征，同时引入“重点特征选择”机制，使模型在特征输入到分类头之前，能够自动判断哪些局部区域对最终预测最为关键，并仅依赖这些核心特征完成分类。通过这一方式，我们期望模型能够在手写稿出现缺角、污渍遮挡、书写不规范等情况下依然保持稳健表现。

最终，我们希望实现一种鲁棒、轻量且可解释性更高的手写字迹识别模型：即便在手写材料质量不佳的情况下，也能够捕捉最具辨识度的笔画特征，从而保持可靠的识别效果。这将为今后在司法审查、身份核验、智能文档处理等领域的落地应用提供坚实的技术支撑。

##### 2025.11.15 记录

1、开始的时候，让ai写了一个很简单的网络（3层卷积池化+全连接），但是效果不理想，准确率大概只有40%（一共就3类，和全蒙差不多了），排查了一下，发现过拟合了（测试集上几乎全队，训练集上40%）

2、调了个参数试试：把卷积多加了两层（相当于减少了中间高维特征的维度，使模型只关注重要的影响特征，不要对那些没用的过拟合），提升很大，准确率大概有60%了

3、继续调参虽然还能有提升，但是现在暂时没有必要了，接下来优先做：

   - 洗数据，同时扩大数据规模
   - 改模型结构
   - 数据增强

4、这只算是一个测试可行性的代码，我们还是要确认一下是进行这样的分类任务，还是类似人脸识别那种的匹配任务。

##### 2025.11.16 记录

1、现在的网络结构是 Encoder + Selector + Classifier

- Encoder：
  - 4个卷积块 (B, 1, 128, 128) -> (B, 32, 64, 64) -> (B, 64, 32, 32) -> (B, 128, 16, 16) -> (B, 256, 8, 8) 
    - Conv 3x3
    - BatchNorm
    - ReLU
    - MaxPool(2x2)
  - 自适应平均池化 (B, 256, 8, 8) -> (B, 256, 1, 1) -> (B, 256)
  - 全连接 (B, 256) -> (B, 256) -> (B, 128)
  - L2 归一化 (B, 128)
- Selector
  - 接收 Encoder 卷积后的 feat (B, 256, 8, 8)
  - 卷积块1 (B, 256, 8, 8) -> (B, 512, 4, 4)
    - Conv 3x3
    - BatchNorm
    - ReLU
    - MaxPool(2x2)
  - 卷积块2 (B, 512, 4, 4) -> (B, 1, 4, 4)
    - Conv 1x1
  - 展平 (B, 1, 4, 4) -> (B, 16)
  - Softmax 归一化
  - Gumbel 采样，保证可导
  - reshape 回空间图 (B, 16) -> (B, 1, 4, 4)
  - 上采样回原尺寸 (B, 1, 4, 4) -> (B, 1, 128, 128)
- Classifier
  - LInear(128, 128)
  - ReLU
  - Linear(128, 3)
